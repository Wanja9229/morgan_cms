# Morgan Edition - 프로젝트 개요

> 자캐 커뮤니티 특화형 CMS
> 기반: 그누보드5 + Tailwind CSS + Supabase

---

## 1. 프로젝트 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Morgan Edition (가칭) |
| 목적 | 자캐(자작 캐릭터) 커뮤니티 운영을 위한 특화형 CMS |
| 기반 | 그누보드5 최신 버전 |
| 레거시 참고 | vn_engine (아보카도) - 모던화 리팩토링 |

---

## 2. 기술 스택

### 2.1 백엔드
| 기술 | 버전/비고 |
|------|----------|
| PHP | 8.x (권장) |
| MySQL/MariaDB | 5.7+ / 10.2+ |
| 그누보드5 | 최신 버전 기반 커스터마이징 |

### 2.2 프론트엔드
| 기술 | 용도 |
|------|------|
| Tailwind CSS | UI 스타일링 |
| Vanilla JS (fetch API) | 세미 실시간 통신 |

### 2.3 실시간 통신
| 기술 | 용도 |
|------|------|
| Supabase | 실시간 기능이 필요한 부분 연동 (채팅, 알림 등 예정) |

### 2.4 개발 환경
| 환경 | 설명 |
|------|------|
| 로컬 | 개발 및 1차 테스트 |
| 실서버 | 배포 테스트 및 운영 |

---

## 3. 아키텍처 개요

### 3.1 페이지 구조
```
┌─────────────────────────────────────────┐
│  Header (PHP 렌더링, 고정)               │
├─────────────────────────────────────────┤
│                                         │
│  Main Content Area                      │
│  ┌─────────────────────────────────┐   │
│  │  fetch 기반 비동기 콘텐츠 로딩    │   │
│  │  - 페이지 전환 없이 콘텐츠 갱신   │   │
│  │  - SPA-like 사용자 경험          │   │
│  └─────────────────────────────────┘   │
│                                         │
├─────────────────────────────────────────┤
│  Footer (PHP 렌더링, 고정)               │
└─────────────────────────────────────────┘
```

### 3.2 통신 방식
- **기본**: 그누보드 PHP 기반 요청/응답
- **본문 영역**: fetch API를 통한 세미 실시간 통신
- **실시간 필요 시**: Supabase Realtime 연동

## 4. 디자인 방향

| 항목 | 내용 |
|------|------|
| 스타일 | 디스코드 참고 |
| 테마 | 다크 모드 고정, 테마 시스템 지원 |
| 메인 컬러 | #f59f0a (황금/앰버) |
| 프레임워크 | Tailwind CSS |
| 적용 범위 | 유저단 (관리자는 고정 스타일) |

### 4.1 디자인 원칙
- 디스코드의 다크 테마 색상 참고
- 좌측 사이드바 네비게이션 (디스코드 서버 목록 스타일 고려)
- 깔끔하고 직관적인 UI
- 모바일 퍼스트 반응형 지원
- 전면 컴포넌트화 (PHP 함수)

### 4.2 테마 시스템
- **기본 테마**: Morgan Edition 기본 (default)
- **확장성**: 추가 테마 설치/판매 가능한 구조
- **원칙**: CSS 변수 기반, 기능/로직은 테마가 건드리지 않음
- **적용 제외**: 관리자 페이지는 항상 기본 스타일

> 상세: [PLAN_UI.md](./PLAN_UI.md)

## 5. 권한 체계

| 레벨 | 명칭 | 권한 |
|------|------|------|
| 1 | 일반유저 | 기본 커뮤니티 이용 |
| 2 | 스태프 | 일부 관리 권한 (게시글 관리, 신고 처리 등) |
| 3 | 관리자 | 전체 시스템 관리 |

> 세부 권한 설계는 별도 문서로 정리 예정

---

## 6. 핵심 기능 목록

### 6.1 회원 시스템
- 간소화된 회원가입 (아이디 + 비밀번호 + 닉네임)
- CAPTCHA 봇 방지
- 이메일/휴대폰 수집 안 함
- 세션 기반 인증

> 상세: [PLAN_MEMBER.md](./PLAN_MEMBER.md)

### 6.2 캐릭터 시스템
- **1계정 다캐릭터** 구조
- 캐릭터 이름 외 항목은 **가변적** (관리자가 프로필 양식 커스터마이징)
- 최초 설치 시 샘플 데이터 자동 삽입
- 캐릭터별 경험치/활동량 관리

> 상세: [PLAN_CHARACTER.md](./PLAN_CHARACTER.md)

### 6.3 게시판 시스템
- **로그 게시판**: 유저 창작물 게시
- **역극 게시판**: 캐릭터 이입 릴레이 소설
- 스킨 기반 확장성 확보
- 추가 게시판 타입은 점진적으로 개발

> 상세: [PLAN_BOARD.md](./PLAN_BOARD.md)

### 6.4 포인트 시스템
- 계정 단위 공용 포인트 (mb_point)
- 커뮤니티 활동으로 적립
- 추후 상점 시스템 연동 예정

## 7. 그누보드 커스터마이징

### 7.1 제거 대상 기능
| 분류 | 제거 항목 |
|------|----------|
| 쇼핑몰 | 상품, 장바구니, 결제, 배송 전체 |
| SNS | 소셜 로그인, 공유 연동 |
| 통계 | 접속자 집계, 로그 분석 |
| 인증 | 본인인증, 이메일 인증, 휴대폰 인증 |
| 기타 | 뉴스레터, SMS 발송, FAQ, 쪽지(필요시 재검토) |

### 7.2 유지/수정 대상 기능
| 분류 | 항목 | 비고 |
|------|------|------|
| 회원 | 가입/로그인/탈퇴 | 간소화 |
| 게시판 | 생성/관리/그룹 | 유지 |
| 포인트 | 적립/차감/내역 | 유지 |
| 관리자 | 기본 설정 | UI 개편 |
| 권한 | 레벨별 권한 | 3단계로 단순화 |

### 7.3 신규 추가 기능
| 분류 | 항목 | 비고 |
|------|------|------|
| 캐릭터 | 프로필/관리 | vn_engine 참고 |
| 캐릭터 | 가변 프로필 양식 | 관리자 커스터마이징 |
| UI | 디스코드 스타일 테마 | Tailwind 기반 |
| 통신 | fetch 기반 비동기 로딩 | SPA-like |
| 실시간 | Supabase 연동 | 추후 적용 |
